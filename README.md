This repository contains my submission for the front-end coding assessment.

```text

README.md
```

## How to Review

### Task 1 - Product Card Component

Location: `task-1/index.html`

- Built as a single self-contained HTML file
- Uses plain HTML, CSS, and JavaScript
- Can be opened directly in a browser

### Task 2 - Interactive Filter Bar

Location: `task-2/`

- Built with plain HTML, CSS, and JavaScript
- Main files are `index.html`, `main.js`, and `style.css`
- Can be opened directly in a browser

### Task 3 - Liquid Snippet

Location: `task-3/`

- `recently-viewed.liquid` contains the Shopify/Liquid snippet
- `task-3/README.md` explains how the snippet would be included in a theme, the chosen approach, and the main limitations
- This task is written as theme code and documentation rather than something that can be run standalone outside Shopify

### Task 4 - Storyblok Schema Design

Location: `task-4/schema.md`

- Contains the proposed schema for the Hero section and Social Proof section
- Includes field definitions, validation rules, structural reasoning, and the two short written answers requested in the brief

### Task 5 - AI Transparency Log

Location: `ai-log.md`

- Documents which AI tools I used
- Includes examples of where AI was useful and where it produced poor or misleading output
- Includes a short reflection on where I would and would not rely on AI in a real project

## Notes

- Tasks 1 and 2 were approached as browser-run deliverables with no build step or framework.
- Task 3 was approached as a coherent Shopify snippet with supporting documentation, as requested in the brief.
- Task 4 was documented in Markdown table form because it felt clearer than raw JSON for both review and discussion.
